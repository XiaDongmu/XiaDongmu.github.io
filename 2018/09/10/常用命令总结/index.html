常用命令总结

发表于 2018-09-10 | 分类于 Linux |

写这篇博客的目的时记录自己常用的命令，不会的时候可以速查，不用每次都百度这么麻烦！

Linux常用命令

Linux文件系统

bin  (binaries)存放二进制可执行文件
sbin  (super user binaries)存放二进制可执行文件，只有root才能访问
etc (etcetera)存放系统配置文件
usr  (unix shared resources)用于存放共享的系统资源
home 存放用户文件的根目录
root  超级用户目录
dev (devices)用于存放设备文件
lib  (library)存放跟文件系统中的程序运行所需要的共享库及内核模块
mnt  (mount)系统管理员安装临时文件系统的安装点
boot 存放用于系统引导时使用的各种文件
tmp  (temporary)用于存放各种临时文件
var  (variable)用于存放运行时需要改变数据的文件

查看内存free

查看进程

ps -aux |grep firefox(grep的作用是查询字符串)
jps：jps是查询java虚拟机进程

杀死进程

kill -9 PID

追加文件的内容

cat id_rsa.pub >> authorized_keys
（意思是追加id_rsa.pub中的内容到authorized_keys文件的最后）

rz、sz

init 6和reboot

重启，init 6应该时不能重启网络配置的，尽量多用init 6
reboot要少使用

top

查询cpu使用率

查看文件的大小

du -sh file(查询file的大小)
du -sh *（列出当前目录中所有文件的大小）
du -sh    (列出当前目录总大小)

网络配置

编辑网卡详细配置（IP、DNS等）
vi /etc/sysconfig/network-scripts/ifcfg-eth0

查看有几块网卡的详细信息，比如eth0,eth1
rm -rf /etc/udev/rules.d/70-persistent-net.rules

重启网络配置
service network restart

关闭防火墙

1.永久有效：
    开启： chkconfig iptables on 
    关闭： chkconfig iptables off
2.即刻生效
    开启： service iptables start 
    关闭： service iptables stop 
3.开启部分端口
    vim /etc/sysconfig/iptables

添加想要开启的相关端口

-A INPUT -m state --state NEW -m tcp -p tcp --dport 6379 -j ACCEPT
-A INPUT -m state --state NEW -m tcp -p tcp --dport 8080 -j ACCEPT
-A INPUT -m state --state NEW -m tcp -p tcp --dport 8081 -j ACCEPT
-A INPUT -m state --state NEW -m tcp -p tcp --dport 8082 -j ACCEPT
-A INPUT -m state --state NEW -m tcp -p tcp --dport 3306 -j ACCEPT

4.最后:service iptables restart

Vim常用命令

编辑命令

i：光标前编辑，I：光标置于当前行的第一个非空白字符处并进入编辑模式
a:光标后编辑，A:在行尾编辑
o:在光标所有行下行开启新行并编辑
O：在光标所有行上行开启新行并编辑

yy复制该行，yyp复制该行，并粘贴到下一行

光标的移动

h        向左

j        向下

k        向上

l        向右（hjkl记忆方法：左下右上）

w        移动到下一个字的开始处

W        也是移动到下一个字，不过判断标准是空白（意思是遇见on-line这种小写的w会移动两次，但        是大写的W只会移动一次）

b        移动到上一个字的开始处

B        移动到上一个字的开始处，区别同上

e        移动到当前字的末端

E        移动到当前字的末端，区别同上

0移动到行首，$移动到行尾，^移动到行首的非空白字符

删除

dd       删除当前行
dw       删除一个字(从字的光标位置删除到字尾,包括字后空白)

daw      删除一个字(从字首删除到字尾,包括字后空白)

diw      删除一个字(从字首删除到字尾,不包括字后空白)

d0       删至行首

D        删除到行尾

与数字组合

50l        向后移50个单元

d5w        删除后面5个字

dL         删除到屏幕底端

100gg      跳到文件第100行

其余指令

.        重复上一个改变，包括插入，删除，或替换操作

u        撤销

J        把下一行连接到当前行, 实际操作为删除当前行的换行符

>>       向右缩进, 3>>

<<       向左缩进 

CTRL-f   向前翻页(即光标向下移动)

CTRL-b   向后翻页(即光标向上移动)

CTRL-u   向后滚动半屏

CTRL-d   向前滚动半屏

zz       使当前位置成为屏幕的中间
ZZ         快速保存并退出vim
ZQ         快速不保存退出vim

文本选择

v        按行选中，以单元为单位，可以组合使用：v%，vL，v$

V        按行选中，以行为单位

CTRL-v   按列选中

选中后可进行操作

u          转成小写  

U          转成大写  

y         拷贝 

>       向右缩进   

<       向左缩进

复制(yank)与粘贴(paste)

y        拷贝指令，拷贝选定内容到缓冲区

Y        拷贝指令，拷贝当前行的内容到缓冲区

p        粘贴指令，粘贴由x, X, d, D, y引起的缓冲区

P        粘贴指令，不同的是p把缓冲区内容放在当前光标之后

yy       拷贝当前行

y4j      拷贝当前行和后 4 行

ddp      对换当前行和下一行

字符替换

rx        把当前字符替换为x, x为任意字符

R         替换n个字符, 直到按Esc或^[为止

字串查找

/        向后查找

?        向前查找

*        向后查找光标所在字

#        向前查找光标所在字

n        下一个

N        上一个

大小写转换

~        大小写互换

g~~      把该行的大小写互换

guu      把当前行换成小写    

gUU      把当前行换成大写  

gUw      把当前光标位置一直到当前字结束换成大写

对码农比较有用的东西

%        在()，[]，{}之间移动
==         格式化
[[       移动到一个代码块的开头
][       移到到一个代码块的结尾

Git常用命令

Git 使用过程中用的命令

$ git add readme.txt
$ git commit -m "wrote a readme file"
$ git status        
$ git diff readme.txt

git log查看提交历史，以确定回退到哪个版本
$ git log
$ git log --pretty=oneline

重返未来,git reflog查看将来的版本号，原因是将来的版本号git log看不见
$ git reflog
$ git reset --hard HEAD^（head是当前版本，回退到上一个版本）
$ git reset --hard 3628164

工作区与暂存区：

    前面讲了我们把文件往Git版本库里添加的时候，是分两步执行的： 
第一步是用git add把文件添加进去，实际上就是把文件修改添加到暂存区； 
第二步是用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支。 

因为我们创建Git版本库时，Git自动为我们创建了唯一一个master分支，所以，现在，git commit就是往master分支上提交更改。

Git丢弃修改：
场景1：当你改乱了工作区某个文件的内容，想直接丢弃工作区的修改时，用命令git checkout -- file。 
场景2：当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步用命令git reset HEAD file，就回到了场景1，第二步按场景1操作。 
场景3：已经提交了不合适的修改到版本库时，想要撤销本次提交，参考版本回退一节，不过前提是没有推送到远程库。

Git删除的对错：
在文件管理器中rm test.txt之后出现两种情况：
（1）删的是对的，那么$ git rm test.txt，彻底删除
（2）删除的是错的，那么$ git checkout -- test.txt git checkout，其实是用版本库里的版本替换工作区的版本，无论工作区是修改还是删除，都可以“一键还原”。

远程仓库：

（1）要关联一个远程库，使用命令git remote add origin git@server-name:path/repo-name.git；（主要使用场景是新建了一个远程库为空，本地库有内容，想把本地库与远程库关联起来）
（2）关联后，使用命令git push -u origin master第一次推送master分支的所有内容；（只有第一次要加 -u） 
（3）此后，每次本地提交后，只要有必要，就可以使用命令git push origin master推送最新修改；

Git clone命令
 $ git clone git@github.com:michaelliao/gitskills.git

分支：
    Git鼓励大量使用分支： 
        查看分支：git branch 
        创建分支：git branch <name> 
        切换分支：git checkout <name> 
        创建+切换分支：git checkout -b <name> 
        合并某分支到当前分支：git merge <name> 
        删除分支：git branch -d <name>

    git分支合并图命令：
        git log --graph --pretty=oneline --abbrev-commit
    分支合并命令：
        git merge --no-ff -m "merge with no-ff" dev

分支如何推送：
    （1）master分支是主分支，因此要时刻与远程同步；
    （2）dev分支是开发分支，团队所有成员都需要在上面工作，所以也需要与远程同步；
    （3）bug分支只用于在本地修复bug，就没必要推到远程了，除非老板要看看你每周到底修复了几个bug；
     （4）feature（新的功能）分支是否推到远程，取决于你是否和你的小伙伴合作在上面开发。

多人协作：
    （1）查看远程库信息，使用git remote -v；
     （2）本地新建的分支如果不推送到远程，对其他人就是不可见的；
    （3）从本地推送分支，使用git push origin branch-name，如果推送失败，先用git pull抓取远程的新提交；
     （4）在本地创建和远程分支对应的分支，使用git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致；
     （5）建立本地分支和远程分支的关联，使用git branch --set-upstream branch-name origin/branch-name； 
    （6）从远程抓取分支，使用git pull，如果有冲突，要先处理冲突。

Git Bash安装及基本配置

安装



基本配置

(1)打开Git Bash查看电脑上是否已经存在SSH密钥：

输入 cd ~/.ssh

(2)创建新的ssh key:

输入 ssh-keygen -t rsa -C "your_email@youremail.com" 

执行这条命令会如上图提示文件保存路径，然后直接按三次Enter，然后会在 .ssh 目录生产两个文件：
id_rsa和id_rsa.pub

(3)用记事本打开.ssh目录下的id_rsa.pub文件，复制里面的内容；

(4) 复制ssh key到github：打开github网站，点击右上角扳手图标，然后点击左边菜单的 ssh key， 然后右边页面的add ssh key，将复制的内容粘贴到github的key中，title可以不填，直接保存即可。

（这一步是到github的，如果自己有搭建git服务器，则可以直接去服务器上（CentOS）将id_rsa.pub内的
公共秘钥导入到到/home/git/.ssh/authorized_keys文件里，一行一个。）

5、测试 ssh 链接 github：

输入 ssh -T git@github.com

出现Successfully就OK；

6、设置自己的git信息：

输入

git config --global user.name "Firstname Lastname" （此处name可修改也不是
用于登录github的登录名）

git config --global user.email "your_email@youremail.com"

设置自己的git信息即完成安装和设置，可以输入git config --list查看自己的git信
息。

（1）安装好Git工具后，找到桌面或者windows的开始菜单下找到Git Bash 快捷方式

（2）选中Git Bash图标，右键，选中“属性”，打开如图：

（3）去掉–cd-to-home，修改“起始位置”为自定义的git 本地仓库的路径，如：F:\Workspace\KYGitLabel

（4）再次打开Git Bash时已经修改了默认路径：

（5）Git CMD 也如上修改，将快捷方式中路径修改为自定义路径即可，这样每次打开Git Bash 和Git CMD就自己进入本地的Git Repository里面了。
